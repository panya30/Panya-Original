What is Panya Ontology?

Panya is a self-learning knowledge system that automatically:

  • Observes and captures information from conversations
  • Extracts entities and relationships
  • Detects patterns across your knowledge
  • Promotes high-quality knowledge to expert level
  • Decays unused information over time

Philosophy: "Version, Don't Delete" ‚Äî Nothing is ever deleted, only superseded.

L1

Raw

Fresh observations, unprocessed data

Decay: 7 days half-life
‚Üí
L2

Extracted

Decay: 30 days half-life
‚Üí
L3

Synthesized

Multiple sources merged, patterns applied

Decay: 90 days half-life
‚Üí
L4

Core

Expert knowledge, validated truth

Never decays

How to Reach L4 (Expert Level)

  1. Add observations ‚Äî Feed knowledge through conversations or manual input
  2. Run learning loop ‚Äî Process and extract entities
  3. Use the knowledge ‚Äî Access it frequently (usage count matters)
  4. Validate patterns ‚Äî Confirm detected patterns are correct
  5. Wait for promotion ‚Äî High confidence + high usage = automatic promotion

Pattern Types

Co-occurrence

Same entities appearing together frequently

e.g., "TypeScript" + "React" appear together 10x

Temporal

Time-based patterns (daily, weekly cycles)

e.g., High activity on Mondays

Semantic

Similar content/meaning clusters

e.g., 5 docs about "API design"

Contradiction

Conflicting information detected

e.g., "Use REST" vs "Use GraphQL"

Evolution

How knowledge changed over time

e.g., Understanding grew from basic to advanced

Validating Patterns

When a pattern is detected, you can:

  • ‚úì Validate: Confirm it's a real pattern ‚Üí enables synthesis
  • ‚úï Reject: Mark as false positive ‚Üí ignored in future

API Reference

Panya exposes a REST API on http://localhost:3100

Core Endpoints

GET /healthHealth check
GET /statsBrain statistics
GET /search?q=querySearch knowledge
POST /learnExtract learnings from messages

Ontology Endpoints

GET /ontology/statsFull ontology statistics
GET /ontology/levelsKnowledge level stats
POST /ontology/observationsAdd observation
POST /ontology/loop/runRun learning loop
GET /ontology/patternsList patterns
POST /ontology/patterns/detectDetect patterns
GET /ontology/identityGet Robin identity

Example: Add Observation via API

curl -X POST http://localhost:3100/ontology/observations \
  -H "Content-Type: application/json" \
  -d '{"type": "conversation", "content": "TypeScript is great for large codebases"}'

Robin's Identity System

Robin has protected identity facets that define her personality:

personality

Core character traits, first principles thinking

voice

Language style, pronouns, tone

values

Principles and beliefs

relationship

Memory of interactions, milestones, inside jokes

üîí Locked Facets

Core identity facets are locked and cannot be auto-modified. Changes require human validation to protect Robin's personality.
