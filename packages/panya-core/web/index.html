User Guide

Learn how Panya's self-learning ontology system works

What is Panya?

Panya is a self-learning knowledge system that grows your knowledge base automatically.

Key Features

  • Add Observations ‚Äî Feed GitHub repos, conversations, or any content
  • Auto-Discovery ‚Äî Automatically track repos for new commits
  • Learning Loop ‚Äî Process, extract, and synthesize knowledge
  • Knowledge Levels ‚Äî Knowledge evolves from L1 (Raw) ‚Üí L4 (Expert)
  • Pattern Detection ‚Äî Find connections across your knowledge

How It Works

Add Observation ‚Üí Auto-Process ‚Üí Learning Loop ‚Üí Knowledge Grows

First Principle: Knowledge evolves through versioning, not deletion. Old information is superseded, never lost.

How to Reach L4 (Expert Level)

  1. Add observations ‚Äî Feed knowledge through conversations or manual input
  2. Run learning loop ‚Äî Process and extract entities
  3. Use the knowledge ‚Äî Access it frequently (usage count matters)
  4. Validate patterns ‚Äî Confirm detected patterns are correct
  5. Wait for promotion ‚Äî High confidence + high usage = automatic promotion

Pattern Types

Co-occurrence

Same entities appearing together frequently

e.g., "TypeScript" + "React" appear together 10x

Temporal

Time-based patterns (daily, weekly cycles)

e.g., High activity on Mondays

Semantic

Similar content/meaning clusters

e.g., 5 docs about "API design"

Contradiction

Conflicting information detected

e.g., "Use REST" vs "Use GraphQL"

Evolution

How knowledge changed over time

e.g., Understanding grew from basic to advanced

Validating Patterns

When a pattern is detected, you can:

  • Validate: Confirm it's a real pattern ‚Äî enables synthesis
  • Reject: Mark as false positive ‚Äî ignored in future

GitHub Discovery

For GitHub repositories, Panya fetches:

  • Recent commits (SHA, author, message, date)
  • Creates a document for each new commit
  • Tracks last commit SHA to avoid duplicates
Tip: Default check interval is 1 hour. The worker only checks observations with auto-discover enabled.

API Reference

Panya exposes a REST API on http://localhost:3100

Core Endpoints

GET /healthHealth check
GET /statsBrain statistics
GET /search?q=querySearch knowledge
POST /learnExtract learnings from messages

Observation Endpoints

GET /ontology/observationsList observations
POST /ontology/observationsAdd observation (auto-processes)
PUT /ontology/observations/:id/autodiscoverToggle auto-discover
POST /ontology/observations/:id/discoverManual discovery check

Discovery Worker Endpoints

GET /ontology/discovery/statusWorker status
POST /ontology/discovery/startStart worker
POST /ontology/discovery/stopStop worker
GET /ontology/discovery/logsGet activity logs
DELETE /ontology/discovery/logsClear activity logs
POST /ontology/discovery/runRun check now

Learning Loop Endpoints

GET /ontology/statsFull ontology statistics
GET /ontology/levelsKnowledge level stats
POST /ontology/loop/runRun learning loop
GET /ontology/patternsList patterns
POST /ontology/patterns/detectDetect patterns

Example: Add GitHub Repo

curl -X POST http://localhost:3100/ontology/observations \
  -H "Content-Type: application/json" \
  -d '{"type": "github_repo", "content": "https://github.com/owner/repo"}'

Example: Enable Auto-Discovery

curl -X PUT http://localhost:3100/ontology/observations/1/autodiscover \
  -H "Content-Type: application/json" \
  -d '{"enabled": true, "intervalMs": 3600000}'
